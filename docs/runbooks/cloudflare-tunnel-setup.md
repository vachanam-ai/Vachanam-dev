# Cloudflare Tunnel Setup — vachanam-agent

**Owner:** Vinay (manual setup) | **Env:** dev-only | **Reusable by:** future devs
**Last reviewed:** 2026-06-08 | **Execute on:** before Vobiz webhook testing (Task 12)

**Purpose:** expose local FastAPI agent server (`uvicorn agent.server:app --port 7860`) at the stable public URL `https://agent-dev.vachanam.in` so Vobiz webhooks (`/answer`, `/recording-finished`, `/recording-ready`) can reach it during local development. Dev-only — Phase 5 production deployment swaps Cloudflare Tunnel for Fly.io public hostname.

Linked from: `docs/superpowers/specs/2026-06-07-pipecat-vobiz-integration.md` §8.

---

## Prerequisites

- Cloudflare account with `vachanam.in` zone added (DNS managed by Cloudflare). Verify in dashboard: `dash.cloudflare.com` → Websites → `vachanam.in` exists and shows "Active".
- Windows (this dev environment) or any OS supporting `cloudflared`.
- Local agent server running on `localhost:7860` — Task 5 server + Task 9 `/ws` ready before testing webhooks.

---

## 1. Install cloudflared

**Windows (winget):**

```powershell
winget install Cloudflare.cloudflared
```

Verify the install completed:

```powershell
cloudflared --version
```

Expected output: a version string such as `cloudflared version 2024.x.x`. If the command is not found, close and reopen the terminal (winget updates PATH on the process level).

macOS: `brew install cloudflared`
Linux: download from https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/install-and-setup/installation/

---

## 2. Authenticate

```powershell
cloudflared tunnel login
```

This opens a browser window. Log into Cloudflare, select the `vachanam.in` zone, and click Authorize. `cloudflared` writes a certificate to `C:\Users\vinay\.cloudflared\cert.pem`. No manual action needed after the browser step — the CLI prints "You have successfully logged in." and exits.

Do not check `cert.pem` into the repo.

---

## 3. Create named tunnel

```powershell
cloudflared tunnel create vachanam-agent
```

Output includes:
- The tunnel UUID (a long hex string, e.g. `a1b2c3d4-...`).
- Path to the credentials JSON: `C:\Users\vinay\.cloudflared\<UUID>.json`.

Note the UUID — it is required in the config file (step 5). The credentials JSON is auto-generated by the create command; do not create or edit it manually. Do not commit it to the repo.

---

## 4. Route DNS

```powershell
cloudflared tunnel route dns vachanam-agent agent-dev.vachanam.in
```

This creates a CNAME record in the `vachanam.in` Cloudflare zone:

```
agent-dev.vachanam.in → <UUID>.cfargotunnel.com
```

Verify in the Cloudflare dashboard: `vachanam.in` → DNS → Records → look for `agent-dev` with type CNAME. The record may take up to 60 seconds to appear.

---

## 5. Create config file

Create or overwrite `C:\Users\vinay\.cloudflared\config.yml` with:

```yaml
tunnel: vachanam-agent
credentials-file: C:\Users\vinay\.cloudflared\<TUNNEL_UUID>.json

ingress:
  - hostname: agent-dev.vachanam.in
    service: http://localhost:7860
  - service: http_status:404
```

Replace `<TUNNEL_UUID>` with the exact UUID from step 3 (e.g. `a1b2c3d4-5678-...`).

The trailing `service: http_status:404` catch-all is required by cloudflared — ingress rules must have a fallback rule. Any request to a hostname not listed above gets a 404 at the tunnel level rather than reaching your local server.

---

## 6. Start the tunnel

Open a terminal separate from the uvicorn terminal and run:

```powershell
cloudflared tunnel run vachanam-agent
```

Expected output includes lines like:

```
INF Registered tunnel connection connIndex=0 ...
INF Registered tunnel connection connIndex=1 ...
INF Registered tunnel connection connIndex=2 ...
INF Registered tunnel connection connIndex=3 ...
```

Four connections are normal — cloudflared connects to four Cloudflare edge data centers for redundancy. Keep this terminal open. The tunnel dies when you Ctrl+C.

**Optional — run as a Windows service (persistent across reboots):**

```powershell
cloudflared service install
```

After this, cloudflared starts automatically with Windows. To stop the service: `cloudflared service uninstall`.

Note: the Windows service approach means the tunnel is always running even when uvicorn is not. Vobiz will receive a 502 from cloudflared until uvicorn is also started. This is acceptable for dev — cloudflared itself does not crash.

---

## 7. Verify

Start uvicorn in its own terminal first:

```powershell
uvicorn agent.server:app --port 7860
```

With the tunnel also running (step 6), test from a third terminal:

```powershell
curl https://agent-dev.vachanam.in/health
```

Expected: `{"status":"ok"}` with HTTP 200.

Vobiz webhook endpoint check:

```powershell
curl -X POST https://agent-dev.vachanam.in/answer -H "Content-Type: application/json" -d "{}"
```

Expected: any non-502 response (400 or 422 is fine — it means cloudflared reached uvicorn and uvicorn rejected a malformed request, which is correct behavior before a real Vobiz call).

**If you get 502:** check (a) uvicorn is running on port 7860, (b) cloudflared output shows `Registered tunnel connection` lines, (c) DNS has propagated: `nslookup agent-dev.vachanam.in 1.1.1.1` should return the `cfargotunnel.com` CNAME.

---

## 8. Production note

This setup is dev-only. Phase 5 swaps Cloudflare Tunnel for Fly.io public hostname (`vachanam-agent.fly.dev` or a custom domain mapped via the production Cloudflare zone). When that happens:

1. Update `PUBLIC_URL` in `.env` (and the Render/Fly secret) from `https://agent-dev.vachanam.in` to the Fly.io URL.
2. Update the Vobiz Application config (Answer URL, Recording URL) to the new hostname.
3. Run `cloudflared service uninstall` and `cloudflared tunnel delete vachanam-agent` on the dev machine (section 9).

No other code changes are needed — all webhook paths (`/answer`, `/recording-finished`, `/recording-ready`) are the same.

---

## 9. Teardown

```powershell
# Stop the tunnel (Ctrl+C in the run terminal, or if running as a service:)
cloudflared service uninstall

# Delete the named tunnel from Cloudflare
cloudflared tunnel delete vachanam-agent

# Remove the DNS record manually if you want to drop the CNAME:
# Cloudflare dashboard → vachanam.in → DNS → Records → delete agent-dev CNAME
```

The credentials JSON at `C:\Users\vinay\.cloudflared\<UUID>.json` can be deleted after teardown. The `cert.pem` is reusable for creating new tunnels in the same Cloudflare account.

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `cloudflared` returns 502 to Vobiz | uvicorn not listening on 7860 | Start uvicorn first; confirm with `curl http://localhost:7860/health` |
| 404 for `/health` via public URL | Ingress hostname mismatch in config.yml | Confirm `hostname:` in config.yml matches `agent-dev.vachanam.in` exactly |
| `cloudflared tunnel route dns` fails with "record already exists" | CNAME was created in a prior run | Delete the `agent-dev` CNAME from the Cloudflare dashboard, then retry the route dns command |
| Vobiz delivers webhooks but gets timeout | Tunnel up but uvicorn handler is slow or blocked | Check uvicorn logs for unhandled exceptions; confirm `/answer` is async |
| `cert.pem` expired / tunnel login needed again | cert.pem is older than ~6 months | Re-run `cloudflared tunnel login`; cert renews automatically |
| Multiple `cloudflared` processes | Forgot to stop old terminal or service conflict | `Get-Process cloudflared \| Stop-Process`, then `cloudflared tunnel run vachanam-agent` |
| `winget install` fails — package not found | winget index not updated | Run `winget upgrade --all` first, or download the cloudflared MSI directly from the Cloudflare releases page |
