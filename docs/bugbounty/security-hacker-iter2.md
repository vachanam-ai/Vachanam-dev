## Do NOT weaken RULE 1/2/3/4/5 while fixing

- **RULE 1 (tenant isolation):** R1 must resolve the branch from a TRUSTED carrier-side DID, never the spoofable SIP header — lock the trunk, don't loosen the `!= 1` branch refusal at `agent.py:1129/1154`. #3's minimisation fix must keep the lookup branch-scoped (`find_bookings_by_phone` RULE 1 filter stays).
- **RULE 2 (atomic token):** the R3 slot backstop must be a DB unique/exclusion constraint — never replace the Redis INCR with a DB count for the next number. Keep `uq_token_number_confirmed` for token doctors.
- **RULE 3 (held token dies with the call):** keep `_release_hold` / disconnect DECR-rollback (`test_livekit_rule3_hold_release`) intact when touching booking tools.
- **RULE 4 (calendar write part of booking, notifications never block it):** the #6 calendar-ownership check must not make a booking fail on a calendar mis-config — keep the enqueue-on-fail (`booking_calendar.py`) non-blocking.
- **RULE 5 (branch from the dialed DID, never the caller):** R1/C1 fixes must trust a carrier-validated DID/ingress, not the caller-settable SIP attribute or a spoofable XFF entry.
