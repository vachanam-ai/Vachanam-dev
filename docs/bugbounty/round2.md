## Verified non-issues (checked, actually fine — don't waste time)

- **Two simultaneous voice calls racing for the last slot** (booking_tools.py:514-524): Redis INCR is the serializer — first caller gets `slot_count=1`, second gets `2 > max_per_slot` and DECR-rolls back. The `db_confirmed` pre-check plus `max(slot_count, db_confirmed+1)` closes the restart hole for slots. Atomic as designed (the gap is only when confirm runs WITHOUT assign — see C1).
- **Walk-in vs voice racing the same slot**: `create_walkin` goes through the same `assign_token` Redis INCR, so desk and phone contend on the same key — no window.
- **Double-cascade idempotency** (cascade_cancel.py:76-97, 103-114): marking the same doctor unavailable twice is safe — `ON CONFLICT uq_doctor_unavailability_doctor_date DO NOTHING` + the token select filters `status='confirmed'`, so already-cancelled tokens are not re-cancelled and no duplicate FollowupTasks are created.
- **Cascade transaction boundary**: steps a-d (unavailability rows, cancels, followups) commit atomically; a failure rolls all of them back together — no half-cancelled state. Calendar enqueue is correctly outside (Rule 4).
- **cascade_rebook_caller channel filter**: tasks are created with `channel='whatsapp'` but the caller job filters on `task_type='cascade_rebook'` only — patients DO get called (checked because it looked wrong).
- **expire_on_commit**: jobs commit inside loops over live ORM rows; safe because the session factory sets `expire_on_commit=False` (database.py:52) — no MissingGreenlet refresh on next iteration.
- **Outbound tenant resolution**: entrypoint prefers `meta["branch_id"]` for outbound (agent.py:885-891) before any DID fallback — round-1 B2 fix is real.
- **DID ambiguity**: `len(branches) != 1` aborts the call instead of serving an ambiguous tenant (agent.py:896-906) — correct Rule-1 behavior.
- **Service-blocked gate fail-open** (agent.py:944-945): a gate-check DB error lets the call proceed rather than killing it — correct per Rule 8.
- **assert_branch_access** (branch_guard.py): super_admin/is_admin hard-blocked from every branch-scoped route; org_admin org-inheritance does a live DB check on branch.org_id; receptionist/doctor need explicit branch_ids. No bypass found in any router (all 7 branch-scoped routers call it before touching data).
- **JWT revocation TTL on logout** (auth.py:200-206): TTL is over-estimated (full jwt_expire_hours) — wasteful, not a security hole; the token still dies immediately.
- **OTP brute force**: 5 attempts then the code is burned; `secrets.compare_digest`; per-IP limiter on the route — fine.
- **Queue optimistic UI** (Queue.jsx:107-130): cancel/rollback/invalidate pattern is correct; 409 on already-terminal status prevents silent double-marking server-side.
- **XSS**: no `dangerouslySetInnerHTML` anywhere in frontend/src; all patient/doctor names render through JSX text nodes (React-escaped).
- **Per-call Redis clients in booking_tools/_do_cancel**: deliberate loop-binding mitigation (documented), cost is ms-level — not a leak (each path closes via context manager or `finally: aclose()`).
- **calendar_writer backoff math**: `_compute_next_attempt` indexes attempts 1-4 against a 4-entry table and attempts≥5 goes failed_permanent first — no IndexError path.
- **payments verify HMAC** (payments.py:218-227): `hmac.compare_digest` over the documented `order_id|payment_id` payload — correct primitive; the persistence gap is already TD-019/B10 (not re-reported).
