## Verified non-issues (checked, actually fine — don't waste time)

- **Token counter never DECR'd on cancel** (`_do_cancel`, agent.py:660-681): correct — comment explains the counter IS the queue sequence; only slot keys are released, guarded against going negative. Matches FIXLOG #24.
- **Disconnect token rollback** (`_cleanup_on_shutdown`, agent.py:1004): correctly guarded with `current == state.token_number` so it never blind-DECRs after someone else INCRed.
- **confirm_booking calendar retry** (booking_tools.py:660): retry is scoped to the calendar call only (no function-level @retry), so transient calendar failure no longer duplicates Token rows. Matches FIXLOG #23.
- **Reschedule atomicity** (`_do_reschedule`): new slot is confirmed before old is cancelled; assign-step failure keeps the old booking (test_reschedule_failure_keeps_old_booking covers it).
- **DID uniqueness** (schema.py:71 partial unique index + branches.py:194 collision check): cross-tenant DID hijack is blocked. Matches FIXLOG #7.
- **assign_token appointment path** double-book: `max(slot_count, db_confirmed+1) > max_per_slot` with INCR-then-rollback is correctly atomic under concurrency.
- **Patient name XSS** (Queue.jsx:45): React escapes `{p.patient_name}` by default — no injection.
- **super_admin PII lockout** (branch_guard.py:46, auth_middleware.forbid_admin): DPDP boundary enforced on every branch-scoped route.
- **find_bookings_by_phone** phone-format matching (last-10-digit `LIKE`): handles +91/0/bare formats. Matches FIXLOG #35.
- **JWT revocation** (auth_middleware.py:96): Redis revocation set checked on every request with TTL = remaining exp.
- **calendar delete 404 idempotency** (calendar_service.py:286): treats 404 as success correctly.
